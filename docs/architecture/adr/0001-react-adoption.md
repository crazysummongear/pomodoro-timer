# ADR-0001 React / TypeScript / Vite を採用する

## ステータス

レビュー用ドラフト

## 日付

2026-06-22

## 背景

Pomodoro Timerは、個人利用向けの静的Webアプリケーションとして開発する。

初期リリースではバックエンドを持たず、ブラウザ上でタイマー操作とLocalStorageによる作業履歴保存を行う。

また、本プロジェクトは実務に近い開発プロセスを学ぶことを重視しているため、一般的なフロントエンド開発で利用される構成を採用する必要がある。

## 決定

フロントエンド技術として、React、TypeScript、Viteを採用する。

## 理由

- ReactはコンポーネントベースでUIを分割しやすい。
- TypeScriptにより、作業履歴やタイマー状態の型を明確にできる。
- Viteは開発サーバーとビルドが高速で、学習用途でも扱いやすい。
- Vitest、ESLint、Prettierとの組み合わせが一般的で、CIに組み込みやすい。
- 将来的にNote記事管理やAI記事レビュー機能を追加する場合も、画面やサービスを増やしやすい。

## 代替案

| 代替案 | 採用しない理由 |
| --- | --- |
| Vanilla JavaScript | 実務的な型管理やコンポーネント設計の学習効果が限定的になる。 |
| Vue | 選択肢として妥当だが、指定技術スタックがReactであるため採用しない。 |
| Next.js | 初期リリースは静的SPAで十分であり、ルーティングやSSRの複雑さを持ち込まない。 |

## 影響

- Reactコンポーネント設計が必要になる。
- TypeScriptの型定義とLint設定が必要になる。
- Viteのビルド成果物をS3へ配置するCI/CD設計が必要になる。

## レビュー観点

- 初期リリースの規模に対して妥当な技術選定か
- 将来拡張を妨げないか
- 学習目的に合っているか
