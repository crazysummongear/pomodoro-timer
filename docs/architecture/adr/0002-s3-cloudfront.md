# ADR-0002 AWS S3 / CloudFront で静的配信する

## ステータス

レビュー用ドラフト

## 日付

2026-06-22

## 背景

Pomodoro Timerは、React + Viteでビルドされる静的フロントエンドアプリケーションである。

初期リリースではバックエンドAPIやデータベースを持たないため、静的ファイル配信に適したホスティング構成を採用する。

また、AWSへのデプロイ経験を得ることが本プロジェクトの目的に含まれている。

## 決定

ビルド成果物はAWS S3に配置し、CloudFrontを通じて配信する。

## 理由

- S3は静的ファイルの配置先として適している。
- CloudFrontにより、HTTPS配信、キャッシュ制御、独自ドメイン対応を学習できる。
- GitHub ActionsからS3同期とCloudFrontキャッシュ削除を自動化しやすい。
- 将来、バックエンドや認証を追加する場合も、フロントエンド配信基盤として継続利用できる。

## 代替案

| 代替案 | 採用しない理由 |
| --- | --- |
| GitHub Pages | 手軽だが、AWSデプロイを学ぶ目的に合わない。 |
| Vercel | React/Viteとの相性はよいが、AWS構成の学習目的から外れる。 |
| Amplify Hosting | AWS内の選択肢として妥当だが、S3とCloudFrontの基本構成を学ぶ目的を優先する。 |

## 影響

- S3バケット、CloudFrontディストリビューション、権限設定が必要になる。
- SPAルーティングを行う場合、CloudFrontのエラーレスポンス設定が必要になる可能性がある。
- GitHub ActionsでAWS認証情報またはOIDC設定が必要になる。
- デプロイ手順とロールバック手順をReleaseフェーズで定義する必要がある。

## レビュー観点

- 初期リリースの静的アプリ配信として妥当か
- AWS学習目的に合っているか
- CI/CDで自動化可能な構成か
