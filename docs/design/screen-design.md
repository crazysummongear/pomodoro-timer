## 3. タイマー画面

### 3.1 目的

利用者が作業タイマーを開始・停止・リセットし、保存済みの作業履歴を確認できるようにする。

### 3.2 表示要素

| 要素ID | 要素名 | 内容 |
| --- | --- | --- |
| UI-001 | アプリ名 | `Pomodoro Timer` を表示する。 |
| UI-002 | タイマー表示 | 残り時間を `mm:ss` 形式で表示する。 |
| UI-003 | 状態表示 | `待機中`、`実行中`、`一時停止中`、`完了` のいずれかを表示する。 |
| UI-004 | 開始ボタン | タイマーを開始または再開する。 |
| UI-005 | 停止ボタン | 実行中のタイマーを一時停止する。 |
| UI-006 | リセットボタン | タイマーを初期状態に戻す。 |
| UI-007 | 作業履歴一覧 | 保存済みの作業セッションを新しい順に表示する。 |
| UI-008 | 空状態表示 | 履歴が存在しない場合に空状態を表示する。 |

### 3.3 状態別の操作可否

| 状態 | 開始 | 停止 | リセット |
| --- | --- | --- | --- |
| idle | 有効 | 無効 | 無効 |
| running | 無効 | 有効 | 有効 |
| paused | 有効 | 無効 | 有効 |
| completed | 有効 | 無効 | 有効 |

## 5. レスポンシブ方針

| 画面幅 | 方針 |
| --- | --- |
| PC | タイマー操作と履歴を縦方向に配置し、中央寄せの読みやすい幅にする。 |
| タブレット | PCと同様の構成を維持し、余白を調整する。 |
| モバイル | ボタンが押しやすい幅になるよう縦または折り返し配置にする。 |

## 6. アクセシビリティ方針

- ボタンには明確なラベルを設定する。
- タイマー状態は色だけでなくテキストでも表示する。
- キーボード操作で主要ボタンにフォーカスできるようにする。
- 履歴一覧は日時、時間、状態が読み取れる構造にする。

## 7. レビュー観点
